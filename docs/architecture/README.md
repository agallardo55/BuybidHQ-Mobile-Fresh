# Architecture Documentation

This directory contains system design, data models, security practices, and operational documentation.

## Purpose

Architecture documentation covers:
- System design and data models
- Security best practices
- Migration procedures
- Deployment processes
- Code quality improvements

## Files

- `data-model.md` - Database schema and data model documentation
- `security-best-practices.md` - Security guidelines and practices
- `MIGRATION_INSTRUCTIONS.md` - Database migration procedures
- `DEPLOYMENT_CHECKLIST.md` - Deployment procedures and checklist
- `IMPROVEMENTS.md` - Code quality improvements and architectural changes

## Workflow

- Review architecture docs before making schema changes
- Update data model docs when modifying database structure
- Follow security practices when implementing features
- Consult migration instructions before running migrations
- Use deployment checklist for production deployments
